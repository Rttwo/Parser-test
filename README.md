# 🤖 Solana Arbitrage Bot — Jupiter vs MEXC

Бот следит за ценами 20 токенов Solana.  
Если цена на DEX (Jupiter) и MEXC расходится на 3%+ — приходит уведомление в Telegram.

**Работает 24/7 бесплатно в облаке через Railway.**

---

# 📋 Что нужно сделать (обзор)

1. Создать аккаунт на GitHub и залить туда файлы бота
2. Создать Telegram-бота и узнать свой ID
3. Создать аккаунт на Railway и подключить GitHub
4. Указать секреты (токен бота, твой ID) в Railway
5. Нажать Deploy — бот запустится сам

Всё займёт ~20 минут. Поехали!

# 🪜 Подробная инструкция

---

## ШАГ 1 — Создай аккаунт на GitHub

GitHub — это место где хранится код бота.

1. Зайди на **https://github.com**
2. Нажми **Sign up** (зарегистрироваться)
3. Введи email, придумай пароль и username
4. Подтверди email

---

## ШАГ 2 — Создай репозиторий на GitHub

Репозиторий — это папка с твоим проектом на GitHub.

1. После входа на GitHub нажми зелёную кнопку **"New"** (или плюс + вверху справа → New repository)
2. В поле **Repository name** напиши: `solana-arb-bot`
3. Выбери **Private** (приватный — только ты видишь)
4. Нажми **Create repository**

---

## ШАГ 3 — Загрузи файлы на GitHub

1. На странице только что созданного репозитория нажми **"uploading an existing file"** (ссылка в тексте по центру страницы)
2. Перетащи в браузер ВСЕ эти файлы из папки с ботом:
   - `bot.py`
   - `tokens.py`
   - `requirements.txt`
   - `runtime.txt`
   - `Procfile`
   - `.gitignore`
   - `README.md`

   ⚠️ **НЕ загружай** файл `config.py` — там будут твои секреты!

3. Внизу страницы нажми зелёную кнопку **"Commit changes"**
4. Файлы появятся в репозитории — готово!

---

## ШАГ 4 — Создай Telegram-бота и узнай свой Chat ID

### Создай бота:
1. Открой Telegram, найди **@BotFather** (синяя галочка ✓)
2. Напиши: `/newbot`
3. Он спросит **имя** — напиши любое, например: `Арбитражный бот`
4. Он спросит **username** — должен заканчиваться на `bot`, например: `my_sol_arb_bot`
5. BotFather пришлёт **токен** — длинная строка вида:
   ```
   7123456789:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   Скопируй и сохрани его — понадобится на шаге 7.

### Узнай свой Chat ID:
1. Найди в Telegram бота **@userinfobot**
2. Напиши ему что угодно (например `/start`)
3. Он ответит числом — например `987654321`
4. Скопируй это число — понадобится на шаге 7.

---

## ШАГ 5 — Создай аккаунт на Railway

Railway — это облачный сервис где будет работать бот.

1. Зайди на **https://railway.app**
2. Нажми **"Start a New Project"** или **Login**
3. Выбери **"Login with GitHub"** — это важно, Railway подключится к твоему GitHub
4. Разреши Railway доступ к GitHub (нажми Authorize)
5. Введи номер телефона для верификации (Railway требует это для бесплатного плана)

> 💡 Бесплатный план Railway даёт $5 кредитов в месяц. Этот бот потребляет ~$0.50-1/месяц — то есть фактически **бесплатно**.

---

## ШАГ 6 — Создай проект в Railway

1. На главной Railway нажми **"New Project"**
2. Выбери **"Deploy from GitHub repo"**
3. Найди в списке `solana-arb-bot` и нажми на него
4. Railway начнёт анализировать код — подожди 10-20 секунд
5. Появится страница проекта — пока не нажимай Deploy!

---

## ШАГ 7 — Добавь секреты в Railway

Это самый важный шаг. Здесь ты вводишь токен бота и свой ID — они хранятся в Railway и никогда не попадают на GitHub.

1. На странице проекта в Railway нажми на карточку сервиса (прямоугольник с названием бота)
2. Перейди на вкладку **"Variables"** (переменные)
3. Нажми **"New Variable"** и добавь по одной:

| Название (Name) | Значение (Value) |
|---|---|
| `TELEGRAM_TOKEN` | токен от @BotFather (длинная строка с цифрами и буквами) |
| `CHAT_ID` | твой ID от @userinfobot (просто число) |
| `THRESHOLD_PERCENT` | `3.0` |
| `CHECK_INTERVAL` | `30` |

Для каждой переменной:
- Нажми **"New Variable"**
- В левое поле введи название (например `TELEGRAM_TOKEN`)
- В правое поле вставь значение
- Нажми галочку ✓ чтобы сохранить

После добавления всех 4 переменных должно выглядеть примерно так:
```
TELEGRAM_TOKEN    = 7123456789:AAFxxx...
CHAT_ID           = 987654321
THRESHOLD_PERCENT = 3.0
CHECK_INTERVAL    = 30
```

---

## ШАГ 8 — Запусти бота!

1. Перейди на вкладку **"Settings"** в Railway
2. Найди раздел **"Service"** → **"Start Command"**
3. Убедись что там написано `python bot.py` (или оно подхватится автоматически из Procfile)
4. Перейди на вкладку **"Deployments"**
5. Нажми **"Deploy"** (или он уже задеплоился автоматически)

Railway скачает код с GitHub, установит библиотеки и запустит бота.

**Через 1-2 минуты** в Telegram от твоего бота придёт сообщение:
```
🤖 Бот арбитража запущен

📋 Токенов: 20
⏱ Интервал: каждые 30 сек
🎯 Порог: 3.0%

Слежу за Jupiter vs MEXC...
```

**Всё! Бот работает 24/7 в облаке** 🎉

2. Railway автоматически увидит изменение и перезапустит бота через 1-2 минуты

## ❓ Частые проблемы

**Railway показывает ошибку "Build failed"**
→ Проверь что все файлы загружены на GitHub, особенно `requirements.txt` и `Procfile`

**Бот задеплоился но сообщение в Telegram не пришло**
→ Проверь переменные в Railway: Variables → убедись что TELEGRAM_TOKEN и CHAT_ID введены верно (без лишних пробелов)

**"KeyError: TELEGRAM_TOKEN"**
→ Переменная не добавлена в Railway. Зайди в Variables и добавь её

**Бот остановился через несколько часов**
→ Railway на бесплатном плане иногда засыпает. Зайди в Deployments и нажми Redeploy. Или перейди на план за $5/мес — тогда работает без остановок.

**Токен не найден на MEXC (предупреждение в логах)**
→ Не страшно. Значит этот токен не торгуется на MEXC и просто пропускается.

Всё меняется в Railway → Variables:
